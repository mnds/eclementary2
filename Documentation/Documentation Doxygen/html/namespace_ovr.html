enum Ovr.HmdCaps

HMD capability bits reported by device.

Enumerator
Present 

The HMD is plugged in and detected by the system.

Available 

The HMD and its sensor are available for ownership use. i.e. it is not already owned by another application.

Captured 

Set to 'true' if we captured ownership of this HMD.

ExtendDesktop 

(read only) Means the display driver is in compatibility mode.

NoMirrorToWindow 

Disables mirroring of HMD output to the window. This may improve rendering performance slightly (only if 'ExtendDesktop' is off).

DisplayOff 

Turns off HMD screen and output (only if 'ExtendDesktop' is off).

LowPersistence 

HMD supports low persistence mode.

DynamicPrediction 

Adjust prediction dynamically based on internally measured latency.

NoVSync 

Support rendering without VSync for debugging.

WritableMask 

These bits can be modified by ovrHmd_SetEnabledCaps.

ServiceMask 

These flags are currently passed into the service. May change without notice.

enum Ovr.RenderAPIType

These types are used to hide platform-specific details when passing render device, OS, and texture data to the API.

The benefit of having these wrappers versus platform-specific API functions is that they allow game glue code to be portable. A typical example is an engine that has multiple back ends, say GL and D3D. Portable code that calls these back ends may also use LibOVR. To do this, back ends can be modified to return portable types such as ovrTexture and ovrRenderAPIConfig.
